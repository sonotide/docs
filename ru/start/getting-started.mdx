Добро пожаловать в Sonotide. Это руководство помогает выбрать правильную точку
входа: разрабатывать сам фреймворк из исходников или использовать уже
установленный SDK в своём приложении.

<Note>
  Sonotide активно использует возможности C++20, поэтому вам потребуется современный компилятор. Мы рекомендуем MSVC.
</Note>

## Что вам понадобится

Перед началом убедитесь, что у вас установлены:
- **Windows 10** или **Windows 11**
- **Visual Studio 2022** (или новее) с установленным компонентом «Разработка классических приложений на C++»
- **CMake** (либо встроенный в Visual Studio, либо глобально в переменной `PATH`)

## Выберите стартовый сценарий

- **Вы хотите работать над самим Sonotide**: собирайте репозиторий из исходников
  через пресет `msvc-x64-debug` и используйте примеры и тесты из дерева сборки.
- **Вы хотите использовать Sonotide в своём проекте**: либо соберите install tree
  через `cmake --install`, либо скачайте готовый SDK со страницы
  [GitHub Releases](https://github.com/sonotide/sonotide/releases), а затем
  передайте путь к install root через `CMAKE_PREFIX_PATH`.

Полный сценарий для установленного пакета, включая `find_package(Sonotide CONFIG REQUIRED)`,
описан в [деталях сборки](/ru/start/build).

<Steps>
  <Step title="Сконфигурируйте проект">
    Для начала сгенерируйте кэш CMake с помощью встроенного пресета `msvc-x64-debug`. Этот пресет гарантирует, что все флаги, тесты и примеры будут настроены правильно.
    
    ```bash
    cmake --preset msvc-x64-debug
    ```
    
    <Info>Если `cmake` не находится через `PATH`, используйте бинарник CMake из установленного toolchain.</Info>
  </Step>

  <Step title="Соберите фреймворк">
    Скомпилируйте исходный код библиотеки, тесты и исполняемые примеры.
    
    ```bash
    cmake --build --preset msvc-x64-debug
    ```
  </Step>

  <Step title="Запустите тесты">
    Проверьте ядро фреймворка, запустив встроенные unit-тесты через `ctest`.
    
    ```bash
    ctest --preset msvc-x64-debug
    ```
    
    <Warning>При ручном запуске без пресета обязательно передавайте флаг `-C Debug`, иначе CTest сообщит, что тесты не найдены.</Warning>
  </Step>

  <Step title="Проверьте примеры на практике">
    После успешного прохождения тестов самое время послушать результат! Запустите примеры из папки сборки:

    **Посмотреть список доступных устройств:**
    ```bat
    build\msvc-x64-debug\examples\Debug\sonotide_list_devices.exe
    ```
    
    **Запустить тестовый поток вывода (проверка тишины):**
    ```bat
    build\msvc-x64-debug\examples\Debug\sonotide_render_silence.exe
    ```

    **Проверить высокоуровневую сессию воспроизведения:**
    ```bat
    build\msvc-x64-debug\examples\Debug\sonotide_playback_session.exe "C:\Windows\Media\notify.wav"
    ```
  </Step>
</Steps>

## Что дальше?
Теперь, когда фреймворк успешно работает на вашей машине, вы можете изучить [Ключевые концепции](/ru/core/concepts), чтобы понять архитектуру, или сразу перейти к изучению [модели API](/ru/reference/api).
