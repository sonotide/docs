Ниже описаны самые популярные ошибки и способы их решения.

## Проблемы со сборкой

<AccordionGroup>
  <Accordion title="CTest жалуется, что конфигурация не найдена">
    Убедитесь, что вы явно указали пресет или конфигурацию сборки (`Debug` / `Release`), так как Visual Studio всегда генерирует мульти-конфигурационные проекты.
    
    ```bash
    ctest --preset msvc-x64-debug
    # Или вручную:
    ctest --test-dir build/msvc-x64-debug -C Debug
    ```
  </Accordion>
  <Accordion title="Консоль пишет, что команда CMake не найдена">
    Если у вас установлена Visual Studio, CMake уже есть в системе, просто он не добавлен в системную переменную `PATH`. Вызовите бинарный файл по полному пути, либо просто откройте ярлык "Developer Command Prompt for VS 2022" — там CMake доступен сразу.
  </Accordion>
</AccordionGroup>

## Ошибки во время исполнения (Runtime)

<AccordionGroup>
  <Accordion title="Инициализация Runtime возвращает ошибку 'unsupported_platform'">
    Sonotide — это фреймворк эксклюзивно для **Windows**, и он завязан на подсистемы WASAPI и Media Foundation. Если вы попытаетесь запустить код на macOS или Linux, фреймворк загрузит заглушку и вежливо вернет ошибку `unsupported_platform`.
  </Accordion>
  <Accordion title="Поток успешно открывается, но падает при вызове start()">
    Если `open_render_stream()` отработал успешно, но `start()` возвращает ошибку, это означает, что фоновый поток не смог инициализировать системные ресурсы.
    
    1. Обязательно прочитайте внутренние поля объекта `error`, который вернул метод `start()`. Там будет лежать точный код ошибки ОС (`HRESULT`).
    2. Скорее всего драйвер аудиокарты в последний момент отверг запрошенный формат звука.
  </Accordion>
</AccordionGroup>
