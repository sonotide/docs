## Как читать эту страницу

Здесь собраны публичные классы, структуры и методы из `include/sonotide`. Страница является строгим техническим контрактом между вашим приложением и библиотекой Sonotide.

<Info>
  Если вы только знакомитесь с фреймворком, настоятельно рекомендуем сначала прочитать [Ключевые концепции](/ru/core/concepts) и [Архитектуру](/ru/core/architecture), прежде чем вчитываться в детальные сигнатуры функций.
</Info>

## `runtime.h`

### `struct runtime_options`

Определяет, какие платформенные подсистемы включаются при создании `runtime`.

| Поле | Тип | Значение по умолчанию | Смысл |
| --- | --- | --- | --- |
| `initialize_media_foundation` | `bool` | `true` | Инициализирует Media Foundation во время создания `runtime`. Отключайте только если процесс управляет Media Foundation сам. |
| `enable_mmcss` | `bool` | `true` | Включает MMCSS для рабочих потоков аудио на Windows. Это улучшает планирование real-time задач. |

### `class runtime`

`runtime` — точка входа в Sonotide. Это move-only объект, который владеет
платформенным backend-ом и отвечает за фабрики потоков и playback session.

| Метод | Параметры | Возвращает | Описание |
| --- | --- | --- | --- |
| `static create(runtime_options options = {})` | `options`: настройки создания | `result<runtime>` | Создаёт границу фреймворка. При успехе возвращает готовый `runtime`, при ошибке — `error`. |
| `enumerate_devices(device_direction direction) const` | `direction`: `render` или `capture` | `result<std::vector<device_info>>` | Перечисляет endpoint-ы в указанном направлении. |
| `default_device(device_direction direction, device_role role = device_role::multimedia) const` | `direction`: направление endpoint-а; `role`: роль системного устройства по умолчанию | `result<device_info>` | Возвращает системное устройство по умолчанию для заданного направления и роли. |
| `open_render_stream(const render_stream_config& config, render_callback& callback)` | `config`: настройки вывода; `callback`: объект, который заполняет выходной буфер | `result<render_stream>` | Открывает поток вывода. Поток подготавливается, но не стартует автоматически. |
| `open_capture_stream(const capture_stream_config& config, capture_callback& callback)` | `config`: настройки захвата; `callback`: объект, который потребляет входной буфер | `result<capture_stream>` | Открывает поток захвата. |
| `open_loopback_stream(const loopback_stream_config& config, capture_callback& callback)` | `config`: настройки loopback-захвата; `callback`: объект, который потребляет системный звук | `result<loopback_capture_stream>` | Открывает loopback-поток. |
| `open_playback_session(const playback_session_config& config = {})` | `config`: настройки high-level playback | `result<playback_session>` | Открывает playback session поверх `runtime` и пути вывода. |

### Пример

```cpp
auto runtime_result = sonotide::runtime::create();
if (!runtime_result) {
    return 1;
}

sonotide::runtime runtime = std::move(runtime_result.value());
auto devices_result = runtime.enumerate_devices(sonotide::device_direction::render);
```

## `device_selector.h`

### `struct device_selector`

Описывает, как поток должен выбрать целевой endpoint.

#### `enum class mode`

| Значение | Смысл |
| --- | --- |
| `system_default` | Разрешить системное устройство по умолчанию по направлению и роли |
| `explicit_id` | Привязаться к конкретному `device_id` |

#### Фабричные методы

| Метод | Параметры | Возвращает | Описание |
| --- | --- | --- | --- |
| `system_default(device_direction direction, device_role role = device_role::multimedia)` | `direction`: `render` или `capture`; `role`: роль системного устройства | `device_selector` | Создаёт селектор, который разрешает системное устройство по умолчанию. |
| `explicit_id(device_direction direction, std::string id)` | `direction`: `render` или `capture`; `id`: конкретный идентификатор endpoint-а | `device_selector` | Создаёт селектор, привязанный к конкретному `device_id`. |

#### Поля

| Поле | Тип | Смысл |
| --- | --- | --- |
| `selection_mode` | `mode` | Режим выбора: системное устройство или явный идентификатор |
| `direction` | `device_direction` | Направление endpoint-а |
| `role` | `device_role` | Роль системного устройства, используется только при `selection_mode == mode::system_default` |
| `device_id` | `std::string` | Явный идентификатор устройства, используется только при `selection_mode == mode::explicit_id` |

## Классы-обёртки потоков

### Общий жизненный цикл

`render_stream`, `capture_stream` и `loopback_capture_stream` — move-only
RAII-обёртки. Объект, созданный по умолчанию, пустой. Валидный объект владеет
backend handle потока.

### Общие публичные методы

| Метод | Параметры | Возвращает | Описание |
| --- | --- | --- | --- |
| `is_open() const noexcept` | нет | `bool` | Возвращает `true`, если обёртка сейчас владеет handle потока. |
| `start()` | нет | `result<void>` | Запускает рабочий поток и стартует обработку аудио. |
| `stop()` | нет | `result<void>` | Останавливает рабочий поток. |
| `reset()` | нет | `result<void>` | Возвращает поток в состояние `prepared` после остановки. |
| `close()` | нет | `result<void>` | Закрывает поток окончательно. |
| `status() const` | нет | `stream_status` | Возвращает актуальный снимок состояния. |

### Роли конкретных типов

| Тип | Для чего использовать |
| --- | --- |
| `render_stream` | Вывод аудио через callback заполнения буфера |
| `capture_stream` | Захват микрофона или другого входного источника |
| `loopback_capture_stream` | Захват output-а render-устройства |

### `class playback_session`

`playback_session` — high-level API для воспроизведения поверх `runtime`,
Media Foundation, render path и встроенного эквалайзера.

| Метод | Параметры | Возвращает | Описание |
| --- | --- | --- | --- |
| `is_open() const noexcept` | нет | `bool` | Возвращает `true`, если сессия всё ещё владеет runtime-состоянием. |
| `load(std::string source_uri)` | `source_uri`: локальный путь или URI источника | `result<void>` | Загружает новый источник и подготавливает воспроизведение. |
| `play()` | нет | `result<void>` | Стартует или возобновляет воспроизведение. |
| `pause()` | нет | `result<void>` | Ставит воспроизведение на паузу, не выгружая источник. |
| `seek_to(std::int64_t position_ms)` | `position_ms`: целевая позиция в миллисекундах | `result<void>` | Выполняет seek внутри текущего источника. |
| `set_volume_percent(int volume_percent)` | `volume_percent`: пользовательская громкость в процентах | `result<void>` | Меняет громкость воспроизведения. |
| `set_equalizer_enabled(bool enabled)` | `enabled`: активна ли EQ-обработка | `result<void>` | Включает или выключает EQ-обработку. |
| `select_equalizer_preset(equalizer_preset_id preset_id)` | `preset_id`: идентификатор встроенного пресета | `result<void>` | Применяет один из встроенных EQ-пресетов. |
| `set_equalizer_band_gain(std::size_t band_index, float gain_db)` | `band_index`: индекс полосы от нуля; `gain_db`: усиление в dB | `result<void>` | Меняет одну полосу эквалайзера. |

## `result.h`

### `template <typename T> class result`

`result<T>` — основной тип возврата вида “значение или ошибка” в Sonotide.

| Метод | Параметры | Возвращает | Описание |
| --- | --- | --- | --- |
| `static success(T value)` | `value`: успешное значение | `result<T>` | Создаёт успешный результат. |
| `static failure(error failure)` | `failure`: объект ошибки | `result<T>` | Создаёт неуспешный результат. |
| `has_value() const noexcept` | нет | `bool` | Возвращает `true`, если внутри есть значение. |
| `operator bool() const noexcept` | нет | `bool` | Удобная проверка на успех. |
| `value() &` / `value() const&` / `value() &&` | нет | `T&`, `const T&` или `T&&` | Возвращает сохранённое значение. Вызывать только при `has_value() == true`. |
| `error() &` / `error() const&` | нет | `error&` или `const error&` | Возвращает сохранённую ошибку. Вызывать только при `has_value() == false`. |

### `class result<void>`

Специализация для операций, которые возвращают только успех или ошибку.

| Метод | Параметры | Возвращает | Описание |
| --- | --- | --- | --- |
| `static success()` | нет | `result<void>` | Создаёт успешный void-результат. |
| `static failure(error failure)` | `failure`: объект ошибки | `result<void>` | Создаёт неуспешный void-результат. |
| `has_value() const noexcept` | нет | `bool` | Возвращает `true`, если операция завершилась успешно. |
| `operator bool() const noexcept` | нет | `bool` | Удобная проверка на успех. |
| `error() const&` | нет | `const error&` | Возвращает ошибку, если результат неуспешен. |
