title: "Динамический эквалайзер"
description: "Соберите кастомный EQ layout, настраивайте per-band Q, снимайте точную кривую и безопасно двигайте полосы по оси частот."
---

Этот walkthrough нужен приложениям, которым нужен не фиксированный набор ползунков, а маленький EQ-редактор. Цель здесь именно в том, чтобы пользователь мог таскать band не только вверх-вниз, но и влево-вправо по частоте.

В отличие от простых EQ-сниппетов, этот пример делает семь вещей:

1. читает глобальные ограничения по числу полос и частотам;
2. стартует с кастомного дефолтного layout-а, где полос меньше 10;
3. настраивает ширину полос через `q_value`;
4. добавляет новую полосу только если у Sonotide ещё есть свободная ёмкость;
5. двигает существующую полосу внутри диапазона, который вернула сама сессия;
6. снимает точную steady-state кривую для UI;
7. применяет весь layout обратно как один цельный `equalizer_state`.

## Полный код

```cpp
#include <algorithm>
#include <array>
#include <iostream>
#include <string_view>

#include "sonotide/runtime.h"

namespace {

void print_error(std::string_view stage, const sonotide::error& error) {
    std::cerr << stage
              << " failed | category=" << sonotide::to_string(error.category)
              << " code=" << sonotide::to_string(error.code)
              << " operation=" << error.operation
              << " message=" << error.message
              << '\n';
}

void print_bands(const sonotide::equalizer_state& state) {
    std::cout << "bands=" << state.bands.size()
              << " preset=" << sonotide::to_string(state.active_preset_id)
              << " output_gain_db=" << state.output_gain_db
              << " headroom_compensation_db=" << state.headroom_compensation_db
              << '\n';

    for (std::size_t index = 0; index < state.bands.size(); ++index) {
        const auto& band = state.bands[index];
        std::cout << "  [" << index << "]"
                  << " freq=" << band.center_frequency_hz
                  << "Hz gain=" << band.gain_db
                  << "dB q=" << band.q_value
                  << '\n';
    }
}

}  // namespace

int main() {
    auto runtime_result = sonotide::runtime::create();
    if (!runtime_result) {
        print_error("runtime::create", runtime_result.error());
        return 1;
    }

    sonotide::runtime runtime = std::move(runtime_result.value());

    auto session_result = runtime.open_playback_session();
    if (!session_result) {
        print_error("open_playback_session", session_result.error());
        return 1;
    }

    sonotide::playback_session session = std::move(session_result.value());

    auto state = session.equalizer_state();
    state.enabled = true;
    state.output_gain_db = -2.0f;

    const auto count_limits = sonotide::supported_equalizer_band_count_limits();
    const auto frequency_limits = sonotide::supported_equalizer_frequency_limits();

    std::cout << "band_count range: "
              << count_limits.min_band_count
              << ".." << count_limits.max_band_count
              << '\n';
    std::cout << "frequency range: "
              << frequency_limits.min_frequency_hz
              << "Hz.."
              << frequency_limits.max_frequency_hz
              << "Hz spacing>="
              << frequency_limits.min_band_spacing_hz
              << "Hz\n";

    state.bands = sonotide::make_default_equalizer_bands(5);
    state.bands[0].gain_db = 3.0f;
    state.bands[0].q_value = 0.9f;
    state.bands[1].gain_db = 1.5f;
    state.bands[1].q_value = 1.1f;
    state.bands[2].gain_db = -1.0f;
    state.bands[2].q_value = 1.8f;
    state.bands[3].gain_db = 2.0f;
    state.bands[3].q_value = 2.4f;
    state.bands[4].gain_db = 1.0f;
    state.bands[4].q_value = 1.2f;

    if (state.bands.size() < count_limits.max_band_count) {
        state.bands.push_back(sonotide::equalizer_band{
            .center_frequency_hz = 2400.0f,
            .gain_db = 2.5f,
            .q_value = 2.0f,
        });
    }

    const auto q_limits = sonotide::supported_equalizer_q_limits();
    state.bands[2].q_value = std::clamp(
        state.bands[2].q_value,
        q_limits.min_q_value,
        q_limits.max_q_value);

    auto apply_result = session.apply_equalizer_state(state);
    if (!apply_result) {
        print_error("apply_equalizer_state(initial)", apply_result.error());
        return 1;
    }

    auto move_range = session.equalizer_band_frequency_range(2);
    if (move_range.has_value()) {
        const float requested_frequency_hz = 900.0f;
        const float clamped_frequency_hz = std::clamp(
            requested_frequency_hz,
            move_range->min_frequency_hz,
            move_range->max_frequency_hz);
        auto move_result = session.set_equalizer_band_frequency(2, clamped_frequency_hz);
        if (!move_result) {
            print_error("set_equalizer_band_frequency", move_result.error());
            return 1;
        }
    }

    auto add_result = session.add_equalizer_band(5200.0f, 1.25f);
    if (!add_result) {
        print_error("add_equalizer_band", add_result.error());
        return 1;
    }

    auto gain_result = session.set_equalizer_band_gain(0, 4.0f);
    if (!gain_result) {
        print_error("set_equalizer_band_gain", gain_result.error());
        return 1;
    }

    auto q_result = session.set_equalizer_band_q(0, 2.6f);
    if (!q_result) {
        print_error("set_equalizer_band_q", q_result.error());
        return 1;
    }

    auto remove_result = session.remove_equalizer_band(1);
    if (!remove_result) {
        print_error("remove_equalizer_band", remove_result.error());
        return 1;
    }

    const std::array<float, 7> ui_frequencies_hz{
        31.5f, 63.0f, 125.0f, 500.0f, 1000.0f, 4000.0f, 12000.0f};
    auto curve_result = sonotide::sample_equalizer_response(
        session.equalizer_state(),
        48000.0f,
        ui_frequencies_hz);
    if (!curve_result) {
        print_error("sample_equalizer_response", curve_result.error());
        return 1;
    }

    for (const auto& point : curve_result.value().points) {
        std::cout << "curve " << point.frequency_hz
                  << "Hz -> " << point.response_db
                  << "dB\n";
    }

    print_bands(session.equalizer_state());
    return 0;
}
```

## Что происходит по шагам

### 1. Пример всё ещё стартует через `runtime` и `playback_session`

Эквалайзер здесь не отдельный DSP-helper. Он является частью playback pipeline Sonotide, поэтому пример сначала создаёт [`runtime`](/ru/reference/api#api-runtime), а затем открывает [`playback_session`](/ru/reference/api#api-playback).

Это важно, потому что состояние EQ живёт рядом с transport state, routing-ом и метаданными согласованного audio path.

### 2. `supported_equalizer_*_limits()` даёт UI правила заранее

Пример читает:

- `supported_equalizer_band_count_limits()`;
- `supported_equalizer_frequency_limits()`.

Именно на этот контракт должен опираться ваш UI до того, как он начнёт рисовать handles и принимать drag-жесты.

Вам не нужно хардкодить:

- что максимум полос равен 10;
- что минимальная частота равна 20 Гц;
- что между соседними полосами нужен минимальный зазор.

### 3. `make_default_equalizer_bands(5)` даёт музыкально вменяемую стартовую раскладку

Вместо того чтобы придумывать частоты с нуля, пример просит у Sonotide дефолтный layout на пять полос.

Это полезно, когда:

- вам нужен более простой редактор, чем полный layout по умолчанию;
- вы всё ещё хотите частоты, распределённые в духе встроенной модели Sonotide;
- вы хотите, чтобы встроенные пресеты позже корректно проецировались на ваш кастомный layout.

### 4. `q_value` даёт UI контроль над шириной полосы, а не только над высотой

Пример меняет `q_value` и в исходном snapshot-е, и позже через `set_equalizer_band_q(...)`.

Это важно для FL-подобных или параметрических редакторов, потому что две полосы с одинаковыми:

- центральной частотой;
- gain;

всё равно могут звучать по-разному, если одна широкая, а другая узкая.

`supported_equalizer_q_limits()` сообщает UI безопасный диапазон, а `default_equalizer_q_value` сохраняет старое историческое поведение Sonotide для дефолтной раскладки.

### 5. `apply_equalizer_state(...)` удобнее всего для полной замены layout-а

Пример сначала правит `state.bands`, а потом применяет полный snapshot одним вызовом.

Это правильный паттерн, когда:

- пользователь изменил сразу несколько полос;
- UI импортировал сохранённый пресет из настроек приложения;
- вы перестроили layout после переключения режима вроде “simple EQ” и “advanced EQ”.

Sonotide при этом нормализует состояние, сортирует layout по частотам, clamp-ит gain-ы и пересчитывает производные метаданные.

### 6. `equalizer_band_frequency_range(...)` и есть база для горизонтального drag-а

Самый важный вызов для FL-подобного редактора здесь такой:

```cpp
const auto move_range = session.equalizer_band_frequency_range(2);
```

Этот диапазон уже учитывает:

- глобальные минимумы и максимумы по частоте;
- минимальный зазор между соседними полосами;
- текущий live layout.

Именно его UI должен использовать, когда пользователь тащит band влево или вправо. Тогда `set_equalizer_band_frequency(...)` становится просто обновлением состояния, а не догадкой.

### 7. `preview_equalizer_response(...)` нужен UI для черновой preview-кривой

Если приложение держит локальный draft, пока пользователь таскает ручки,
соберите временный `equalizer_preview_state` и запросите preview через session:

```cpp
sonotide::equalizer_preview_state preview_state;
preview_state.enabled = true;
preview_state.output_gain_db = 2.0f;
preview_state.bands = draft_bands;

auto preview_curve_result =
    session.preview_equalizer_response(preview_state, ui_frequencies_hz);
```

Так вы получите точную кривую, которую Sonotide применил бы после
нормализации, но живая сессия останется неизменной.

### 8. `sample_equalizer_response(...)` нужен UI для точной уже применённой кривой

После редактирования состояния пример вызывает:

```cpp
auto curve_result = sonotide::sample_equalizer_response(
    session.equalizer_state(),
    48000.0f,
    ui_frequencies_hz);
```

Это рекомендуемый offline/editor сценарий, когда вы уже знаете sample rate, под которую хотите смоделировать EQ.

Возвращённые точки уже включают в себя:

- реальный biquad-отклик всех полос;
- текущий `q_value` каждой полосы;
- `headroom_compensation_db`;
- `output_gain_db`.

То есть фронту больше не нужно аппроксимировать кривую своими формулами.

### 9. Добавление, правка и удаление полос идут через session-level helpers

После применения начального snapshot-а пример показывает runtime-операции:

- `add_equalizer_band(...)`;
- `set_equalizer_band_gain(...)`;
- `remove_equalizer_band(...)`.

Эти helpers удобны для инкрементальных правок из UI, потому что не нужно пересобирать полный `equalizer_state` на каждый маленький жест.

### 10. `equalizer_state()` остаётся единственным источником правды после коммита изменений

В конце пример печатает `session.equalizer_state()`.

Именно этому snapshot-у ваш UI должен доверять в вопросах:

- фактического порядка полос;
- текущих gain-ов и центральных частот;
- активного встроенного пресета или `custom`;
- рассчитанной headroom-компенсации.

Если приложение сохраняет настройки EQ, лучше сохранять именно этот нормализованный snapshot, а не промежуточный локальный draft.
