## Почему этот раздел теперь разбит

Теперь у каждого примера Sonotide есть отдельная страница. Так боковое меню
работает удобнее: можно сразу перейти к нужному сценарию, а не прокручивать
один большой документ.

Примеры здесь делятся на две группы:

- готовые исполняемые примеры из репозитория фреймворка;
- более полные walkthrough-сценарии, которые показывают application-level
  интеграцию вне папки `examples/`.

## Запуск готовых примеров из WSL

```bash
cmd.exe /c build\\msvc-x64-debug\\examples\\Debug\\sonotide_list_devices.exe
cmd.exe /c build\\msvc-x64-debug\\examples\\Debug\\sonotide_render_silence.exe
cmd.exe /c "build\\msvc-x64-debug\\examples\\Debug\\sonotide_playback_session.exe C:\\Windows\\Media\\notify.wav"
```

## С чего начать

- Начинайте со [Списка устройств](/ru/examples/list-devices), если хотите проверить границу `runtime` и метаданные endpoint-ов.
- Начинайте с [Вывода тишины](/ru/examples/render-silence), если у вас уже есть PCM и нужен только output path.
- Начинайте с [Сессии воспроизведения](/ru/examples/playback-session), если нужен готовый playback workflow внутри фреймворка.
- Начинайте с [Проверки output-устройства](/ru/examples/output-probe), если приложение должно валидировать конкретный endpoint.
- Начинайте с [Динамического эквалайзера](/ru/examples/dynamic-equalizer), если в UI нужны перемещаемые полосы, кастомные layout-ы и FL-подобные правки по частоте.
- Начинайте с [Плеера spoken-word](/ru/examples/spoken-player), если вы строите небольшой player вокруг `playback_state`.
