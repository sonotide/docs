Вместо сложного WASAPI, Sonotide предлагает три удобных фасада потоков, которые имеют абсолютно одинаковый жизненный цикл.

<CardGroup cols={3}>
  <Card title="Render (Вывод)" icon="volume-high">
    [`render_stream`](/ru/api#api-streams) отправляет PCM-данные на динамики или в наушники.
  </Card>
  <Card title="Capture (Ввод)" icon="microphone">
    [`capture_stream`](/ru/api#api-streams) получает PCM-данные с микрофона или линейного входа.
  </Card>
  <Card title="Loopback (Захват)" icon="laptop-code">
    [`loopback_capture_stream`](/ru/api#api-streams) записывает системный звук (всё, что играет в Windows).
  </Card>
</CardGroup>

## Универсальный жизненный цикл

Вне зависимости от типа потока, вы управляете ими совершенно одинаковыми функциями:

<Steps>
  <Step title="is_open()">Проверяет, выделены ли системные ресурсы.</Step>
  <Step title="start()">Запускает рабочий поток и инициирует движение аудио.</Step>
  <Step title="stop()">Безопасно останавливает передачу звука.</Step>
  <Step title="reset()">Сбрасывает согласованный формат и состояние.</Step>
  <Step title="close()">Уничтожает внутренние хендлы операционной системы.</Step>
</Steps>

<Info>
  Универсальность — важнейший принцип дизайна. Научившись выводить звук через наушники, вы уже умеете писать микрофонный рекордер.
</Info>

## Конфигурация диктует поведение

Чтобы открыть поток, создайте объект конфигурации и передайте его в метод фабрики `runtime`:

```cpp
sonotide::render_stream_config config;
config.device_selector = sonotide::device_selector::default_multimedia();
// При желании здесь можно зафиксировать нужный sample rate

auto stream_result = runtime.open_render_stream(config, my_callback);
```

Базовые поля, общие для всех конфигураций:
- `device_selector`: логика поиска нужного "железа".
- `share_mode` / `callback_mode`: подсказки для интеграции с WASAPI.
- `format_request`: жесткие или плавающие требования к формату (Sample Rate, Channels).
- `auto_recover_device_loss`: политика поведения при отключении устройства (например, выдернули USB кабель).

## Контракт коллбеков

Sonotide берёт на себя тяжёлую работу с потоками, а ваше приложение занимается вычислением байт.

- **Вывод (Render)**: Метод `on_render(...)` получает на вход перезаписываемый `audio_buffer_view`. Ваша задача заполнить его сэмплами.
- **Ввод (Capture)**: Метод `on_capture(...)` получает "read-only" сэмплы от микрофона.
- **Ошибки**: Метод `on_stream_error(...)` позволяет вам асинхронно реагировать на краши (например, отключение наушников).

<Warning>
  Коллбеки выполняются в специальном потоке планировщика с повышенным приоритетом. **Никогда** не блокируйте этот поток (mutex, запись на диск, сеть), иначе в звуке появятся трески и паузы.
</Warning>

## Чтение статуса вместо гадания

Не нужно отправлять опасные COM-запросы к ОС. В любой момент времени вы можете запросить снимок [`stream_status`](/ru/api#api-streams):

```cpp
auto status = my_stream.status();
std::cout << "Текущий стейт: " << (int)status.state << "\n";
std::cout << "Обработано фреймов: " << status.worker_frames_processed << "\n";
std::cout << "Устройство отвалилось? " << status.device_lost << "\n";
```
