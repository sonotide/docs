При работе со звуком в Windows легко утонуть в специфике COM-интерфейсов. Главная задача Sonotide — дать вам понятную ментальную модель, которая отражает то, что вы на самом деле хотите сделать, скрывая системные сложности.

## 1. Разделяйте границы

Главная идея Sonotide: ваше приложение взаимодействует с **финализированным API**, а не с сырыми объектами операционной системы. 

Вы начинаете работу с [`runtime`](/ru/api#api-runtime), описываете конфигурацию и следите за результатом через снимки состояний, такие как [`stream_status`](/ru/api#api-streams).

## 2. Потоки максимально предсказуемы

Звук не должен запускаться или падать "где-то в фоне". Sonotide делает весь жизненный цикл явным:

<Steps>
  <Step title="Open">Создание и подготовка потока в спящем состоянии.</Step>
  <Step title="Start">Явный запуск системной обработки и вызовов callback-уровня.</Step>
  <Step title="Stop">Аккуратная остановка передачи данных.</Step>
</Steps>

<Info>
  Благодаря такому подходу отключение наушников (Device Loss) больше не будет тихой проблемой. Вы можете прочитать `stream_status` и точно узнать, почему поток остановился.
</Info>

## 3. Playback — это надстройка

Базовые потоки отдают или принимают сырые байты. Но многим плеерам нужно загружать файлы, менять маршрутизацию, управлять громкостью и эквалайзером. 

Для этого есть [`playback_session`](/ru/api#api-playback). Это не "другой бэкенд", это оркестратор, построенный поверх тех же базовых потоков Sonotide.

## 4. Парсинг ошибок без исключений

Sonotide не бросает исключений в критическом звуковом тракте. Почти все операции возвращают безопасный объект [`result<T>`](/ru/api#api-error-result).

```cpp
auto runtime_result = sonotide::runtime::create();

if (!runtime_result) {
    const auto& err = runtime_result.error();
    // Вся информация под рукой: err.message, err.operation, err.native_code (HRESULT)
    return;
}

// Если ошибок нет, безопасно извлекаем объект
auto runtime = std::move(runtime_result.value());
```

Такая модель делает краши практически невозможными — компилятор заставит вас проверить статус выполнения функции.
