При использовании высокоуровневой `playback_session` Sonotide предоставляет полностью интегрированный эквалайзер с динамической раскладкой полос. Вы можете оставить привычную дефолтную схему на 10 полос, сократить её до меньшего числа полос или построить свой layout с максимумом в 10 перемещаемых полос.

Так как WASAPI в Windows не имеет таких функций "из коробки", мы вшили алгоритмы напрямую в пайплайн обработки Media Foundation, чтобы вам не пришлось с нуля писать DSP-процессор.

## 1. Базовое использование

Чтобы управлять эквалайзером, вызывайте методы напрямую у вашей сессии воспроизведения:

```cpp
// 1. Включаем обработку
session.set_equalizer_enabled(true);

// 2. Применяем готовый пресет
session.select_equalizer_preset(sonotide::equalizer_preset_id::jazz);

// 3. Немного снижаем общую громкость выхода, чтобы избежать перегруза
session.set_equalizer_output_gain(-1.5f);
```

## 2. Точечная настройка

Если встроенных пресетов (`rock`, `jazz`, `pop`, `spoken_podcast`) недостаточно, вы можете управлять полосами вручную:

```cpp
// Меняем gain существующей полосы
session.set_equalizer_band_gain(0, 3.5f);

// Сдвигаем эту полосу по частоте
session.set_equalizer_band_frequency(0, 72.0f);

// Добавляем новую полосу в low-mid область
session.add_equalizer_band(450.0f, 1.5f);

// Позже можем убрать лишнюю полосу
session.remove_equalizer_band(2);
```

<Warning>
  Агрессивное поднятие полос выше `+0dB` неизбежно вызывает цифровой клиппинг (хруст). Если вы сильно выкручиваете определенные частоты вверх, обязательно компенсируйте это снижением общего `output_gain_db`.
</Warning>

## 3. Ограничения и дефолтные раскладки

EQ теперь динамический, но не бесконечный. Sonotide отдаёт ограничения явно, чтобы ваш UI не гадал:

```cpp
const auto count_limits = sonotide::supported_equalizer_band_count_limits();
const auto frequency_limits = sonotide::supported_equalizer_frequency_limits();

auto state = session.equalizer_state();
if (state.bands.size() < count_limits.max_band_count) {
    session.add_equalizer_band(2200.0f, 0.0f);
}

const auto range = session.equalizer_band_frequency_range(0);
if (range.has_value()) {
    session.set_equalizer_band_frequency(0, range->min_frequency_hz);
}
```

Если нужна готовая раскладка, соберите её через `make_default_equalizer_bands(...)` и примените как полный `equalizer_state`.

## 4. Чтение состояния для интерфейса (UI)

Вы можете в любой момент вытащить всю нужную информацию для отрисовки ползунков в вашем приложении через `session.equalizer_state()`.

`last_nonflat_band_gains_db` удобен, если в интерфейсе нужна кнопка “вернуться к последней настроенной кривой” после временного flat-состояния.

## Примечание к дизайну

Эквалайзер является частью фреймворка, потому что это критично важный функционал для любого медиаплеера. Однако, если вашему приложению нужна сложная обработка со своими компрессорами, ревербом и параметриком, вам следует игнорировать этот слой настройки и применять свою собственную DSP-логику к сырым байтам из обычного `render_stream`.
