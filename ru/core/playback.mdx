Работать с сырыми байтами `render_stream` — отличная идея, если вы пишете свой аудио-движок. Но что, если вы просто хотите сделать музыкальный плеер или встроить звук в приложение? Вам почти наверняка понадобится:
- Загрузить аудиофайл и декодировать его.
- Играть, ставить на Паузу, Перематывать.
- Управлять громкостью и эквалайзером.
- Бесшовно переключать потоки, когда пользователь втыкает наушники.

Чтобы не заставлять вас изобретать велосипед вокруг потоков, в Sonotide встроен класс [`playback_session`](/ru/reference/api#api-playback).

## 1. Настройка сессии

Сессия создается через ту же самую фабрику `runtime`, что и потоки:

```cpp
sonotide::playback_session_config config;
// Автоматически запускать звук сразу после загрузки файла
config.auto_play_on_load = true;
config.initial_volume_percent = 75;

auto session_result = runtime.open_playback_session(config);
if (!session_result) return;

sonotide::playback_session session = std::move(session_result.value());
```

## 2. Управление воспроизведением

Функции сессии прозрачно маппятся на обычные кнопки пользовательского UI:

## 3. Выбор устройства вывода (Routing)

Боль Windows-разработки: управлять тем, куда сейчас "дует" звук. В `playback_session` это встроено:

```cpp
// Показать пользователю список подключенных колонок
auto available_outputs = session.list_output_devices();

// Переключить звук на наушники прямо во время воспроизведения (без пауз)
session.select_output_device(headphones_id);
```

## 4. Чтение стейта воспроизведения

Вам не нужно подписываться на сложную паутину событий, чтобы просто обновлять прогресс-бар в вашем UI интерфейсе. Вы можете создать простой таймер и опрашивать функцию `session.state()`. 

В ответ придёт безопасный иммутабельный слепок [`playback_state`](/ru/reference/api#api-playback) со всеми нужными данными:
- `status`: Сейчас `Playing`, `Paused` или `Failed`?
- `position_ms` / `duration_ms`: На какой мы секунде трека? 
- `active_output_device_name`: Какое устройство в реальности сейчас проигрывает звук?

<Info>
  Используйте `playback_session` всегда, когда вам нужно просто проигрывать готовые аудиофайлы или когда в вашем приложении есть концепция плеера. Опускайтесь на уровень сырых потоков `render_stream` только если вы генерируете PCM-данные самостоятельно.
</Info>
