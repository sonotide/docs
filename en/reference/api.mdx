## How to Read This Reference

This page covers the public headers under `include/sonotide`. It defines the exact contract between your application and the Sonotide framework, including all structures, enums, factory methods, and return types.

<Info>
  If you are new to the framework, it's highly recommended you read the [Core Concepts](/en/core/concepts) and [Architecture](/en/core/architecture) pages first before diving deep into these signatures.
</Info>

## Header map

| Header | Main types | Use it for |
| --- | --- | --- |
| `runtime.h` | `runtime`, `runtime_options` | Creating the framework boundary, enumerating devices, and opening streams or playback sessions |
| `device_info.h`, `device_selector.h` | `device_info`, `device_selector`, device enums | Describing and selecting endpoints |
| `audio_format.h`, `audio_buffer.h` | `audio_format`, `format_request`, `audio_buffer_view` | Format negotiation and callback buffers |
| `stream_config.h`, `stream_callback.h`, `stream_status.h`, `stream_state.h` | stream config types, callbacks, status, state | Configuring and monitoring streams |
| `render_stream.h`, `capture_stream.h`, `loopback_capture_stream.h` | stream wrapper classes | Explicit stream lifecycle control |
| `playback_session.h`, `playback_state.h` | `playback_session`, `playback_state` | High-level playback workflows |
| `equalizer.h` | EQ enums, presets, state helpers | Built-in playback equalization |
| `error.h`, `result.h` | `error`, `result<T>` | Error transport and success-or-failure returns |
| `version.h` | version constants | Reporting the framework version |

## `runtime.h`

### `struct runtime_options`

Controls how the framework initializes platform services when you create a
runtime.

| Field | Type | Default | Meaning |
| --- | --- | --- | --- |
| `initialize_media_foundation` | `bool` | `true` | Initializes Media Foundation during runtime creation. Disable it only if your process already manages Media Foundation explicitly. |
| `enable_mmcss` | `bool` | `true` | Enables MMCSS for stream worker threads on Windows. This improves scheduling for real-time audio work. |

### `class runtime`

`runtime` is the entry point to Sonotide. It is move-only, owns the platform
backend, and is responsible for device discovery and stream/session factories.

| Method | Parameters | Returns | Description |
| --- | --- | --- | --- |
| `static create(runtime_options options = {})` | `options`: runtime creation settings | `result<runtime>` | Creates the framework boundary. On success, you receive a fully initialized `runtime`. On failure, you receive an `error`. |
| `enumerate_devices(device_direction direction) const` | `direction`: `render` or `capture` | `result<std::vector<device_info>>` | Enumerates endpoints in the requested direction. |
| `default_device(device_direction direction, device_role role = device_role::multimedia) const` | `direction`: endpoint direction; `role`: default role to query | `result<device_info>` | Returns the OS default endpoint for the given direction and role. |
| `open_render_stream(const render_stream_config& config, render_callback& callback)` | `config`: render stream settings; `callback`: object that fills output buffers | `result<render_stream>` | Opens a render stream wrapper. The stream is prepared but not started automatically. |
| `open_capture_stream(const capture_stream_config& config, capture_callback& callback)` | `config`: capture settings; `callback`: object that consumes input buffers | `result<capture_stream>` | Opens a capture stream wrapper. |
| `open_loopback_stream(const loopback_stream_config& config, capture_callback& callback)` | `config`: loopback capture settings; `callback`: object that consumes captured system audio | `result<loopback_capture_stream>` | Opens a loopback capture stream wrapper. |
| `open_playback_session(const playback_session_config& config = {})` | `config`: high-level playback settings | `result<playback_session>` | Opens a playback session built on top of the runtime and render path. |

### Usage example

```cpp
auto runtime_result = sonotide::runtime::create();
if (!runtime_result) {
    return 1;
}

sonotide::runtime runtime = std::move(runtime_result.value());
auto devices_result = runtime.enumerate_devices(sonotide::device_direction::render);
```

## `device_selector.h`

### `struct device_selector`

Describes how a stream should pick its target endpoint.

#### `enum class mode`

| Value | Meaning |
| --- | --- |
| `system_default` | Resolve the system default endpoint for a direction and role |
| `explicit_id` | Bind to a specific endpoint identifier |

#### Factory methods

| Method | Parameters | Returns | Description |
| --- | --- | --- | --- |
| `system_default(device_direction direction, device_role role = device_role::multimedia)` | `direction`: render or capture; `role`: default role to use | `device_selector` | Builds a selector that resolves the system default endpoint. |
| `explicit_id(device_direction direction, std::string id)` | `direction`: render or capture; `id`: concrete endpoint identifier | `device_selector` | Builds a selector that binds to a specific endpoint. |

#### Fields

| Field | Type | Meaning |
| --- | --- | --- |
| `selection_mode` | `mode` | Whether selection is by system default or explicit device id |
| `direction` | `device_direction` | Direction of the endpoint |
| `role` | `device_role` | Default-role lookup used only when `selection_mode == mode::system_default` |
| `device_id` | `std::string` | Explicit device identifier used only when `selection_mode == mode::explicit_id` |

## Stream wrapper classes

### Common lifecycle

`render_stream`, `capture_stream`, and `loopback_capture_stream` are move-only
RAII wrappers. A default-constructed wrapper is empty. A valid wrapper owns a
backend stream handle.

### Shared public methods

| Method | Parameters | Returns | Description |
| --- | --- | --- | --- |
| `is_open() const noexcept` | none | `bool` | Returns `true` if the wrapper currently owns a stream handle. |
| `start()` | none | `result<void>` | Starts the worker thread and begins audio processing. |
| `stop()` | none | `result<void>` | Stops the worker thread. |
| `reset()` | none | `result<void>` | Resets the stream back to `prepared` after it has stopped. |
| `close()` | none | `result<void>` | Closes the stream permanently. |
| `status() const` | none | `stream_status` | Returns the current status snapshot. |

### Wrapper-specific roles

| Type | Use it for |
| --- | --- |
| `render_stream` | Output playback callbacks |
| `capture_stream` | Microphone or input capture |
| `loopback_capture_stream` | Recording the output of a render device |

### `class playback_session`

`playback_session` is the high-level playback API built on top of Sonotide
runtime, Media Foundation decoding, the render path, and the built-in
equalizer.

| Method | Parameters | Returns | Description |
| --- | --- | --- | --- |
| `is_open() const noexcept` | none | `bool` | Returns `true` if the session still owns runtime state. |
| `load(std::string source_uri)` | `source_uri`: local path or URI to load | `result<void>` | Loads a new source and prepares playback. |
| `play()` | none | `result<void>` | Starts or resumes playback. |
| `pause()` | none | `result<void>` | Pauses playback while keeping the source loaded. |
| `seek_to(std::int64_t position_ms)` | `position_ms`: target position in milliseconds | `result<void>` | Seeks within the current source. |
| `set_volume_percent(int volume_percent)` | `volume_percent`: user volume from 0 to 100 and above if supported by the implementation | `result<void>` | Changes the playback volume. |
| `set_equalizer_enabled(bool enabled)` | `enabled`: whether EQ processing is active | `result<void>` | Enables or disables EQ processing. |
| `select_equalizer_preset(equalizer_preset_id preset_id)` | `preset_id`: built-in preset identifier | `result<void>` | Applies a built-in EQ preset. |
| `set_equalizer_band_gain(std::size_t band_index, float gain_db)` | `band_index`: zero-based EQ band index; `gain_db`: gain in decibels | `result<void>` | Changes one EQ band. |
| `set_equalizer_band_q(std::size_t band_index, float q_value)` | `band_index`: zero-based EQ band index; `q_value`: band Q value | `result<void>` | Changes the bell width of one EQ band. |
| `add_equalizer_band(float center_frequency_hz, float gain_db = 0.0f)` | `center_frequency_hz`: desired band center in hertz; `gain_db`: initial gain | `result<void>` | Adds a new EQ band when the current layout still has free capacity. |
| `remove_equalizer_band(std::size_t band_index)` | `band_index`: zero-based EQ band index | `result<void>` | Removes one band from the active EQ layout. |
| `set_equalizer_band_frequency(std::size_t band_index, float center_frequency_hz)` | `band_index`: zero-based EQ band index; `center_frequency_hz`: new desired center | `result<void>` | Moves one band left or right within the valid frequency range for that slot. |
| `reset_equalizer()` | none | `result<void>` | Restores a flat EQ curve. |
| `set_equalizer_output_gain(float output_gain_db)` | `output_gain_db`: post-EQ gain in decibels | `result<void>` | Adjusts output gain after EQ processing. |
| `apply_equalizer_state(const equalizer_state& state)` | `state`: full EQ snapshot to apply | `result<void>` | Replaces the current EQ state with a full snapshot. |
| `list_output_devices() const` | none | `result<std::vector<device_info>>` | Lists output devices available to the session. |
| `select_output_device(std::string device_id)` | `device_id`: target output endpoint identifier | `result<void>` | Rebinds playback to another output device. |
| `state() const` | none | `playback_state` | Returns the current playback-state snapshot. |
| `equalizer_state() const` | none | `equalizer_state` | Returns the current EQ-state snapshot. |
| `sample_equalizer_response(std::span<const float> frequencies_hz) const` | `frequencies_hz`: sampling frequencies requested by the caller | `result<equalizer_response_curve>` | Samples the exact steady-state EQ curve for the current live session. |
| `preview_equalizer_response(const equalizer_preview_state& preview_state, std::span<const float> frequencies_hz) const` | `preview_state`: temporary EQ draft to evaluate; `frequencies_hz`: sampling frequencies requested by the caller | `result<equalizer_response_curve>` | Samples the exact curve for a temporary EQ draft without mutating the live session state. |
| `equalizer_band_frequency_range(std::size_t band_index) const` | `band_index`: zero-based EQ band index | `std::optional<equalizer_frequency_range>` | Returns the valid frequency range for moving one active band. |
| `close()` | none | `result<void>` | Closes the session and releases its runtime resources. |

## `result.h`

### `template <typename T> class result`

`result<T>` is the main success-or-error transport type in Sonotide.

| Method | Parameters | Returns | Description |
| --- | --- | --- | --- |
| `static success(T value)` | `value`: successful payload | `result<T>` | Creates a successful result. |
| `static failure(error failure)` | `failure`: error payload | `result<T>` | Creates a failed result. |
| `has_value() const noexcept` | none | `bool` | Returns `true` if a value is present. |
| `operator bool() const noexcept` | none | `bool` | Convenience success check. |
| `value() &` / `value() const&` / `value() &&` | none | `T&`, `const T&`, or `T&&` | Returns the stored value. Call only when `has_value()` is `true`. |
| `error() &` / `error() const&` | none | `error&` or `const error&` | Returns the stored error. Call only when `has_value()` is `false`. |

### `class result<void>`

| Method | Parameters | Returns | Description |
| --- | --- | --- | --- |

| `operator bool() const noexcept` | none | `bool` | Convenience success check. |
| `error() const&` | none | `const error&` | Returns the stored error when the result is a failure. |
