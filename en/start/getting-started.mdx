Welcome to Sonotide. This guide helps you choose the right starting path:
develop the framework from source, or consume an installed SDK from your own
project.

<Note>
  Sonotide relies on C++20 features, so you'll need an up-to-date compiler. We recommend MSVC.
</Note>

## Prerequisites

Before starting, make sure you have the following installed:
- **Windows 10** or **Windows 11**
- **Visual Studio 2022** (or newer) with the Desktop development with C++ workload
- **CMake** (either from Visual Studio or globally available in your `PATH`)

## Choose your starting path

- **You want to work on Sonotide itself**: build the repository from source with
  the `msvc-x64-debug` preset and use the examples and tests from the build tree.
- **You want to use Sonotide in your own project**: either create an install
  tree with `cmake --install` or download a ready-made SDK from the
  [GitHub Releases page](https://github.com/sonotide/sonotide/releases), then
  point `CMAKE_PREFIX_PATH` at that install root.

For the full installed-package flow, including `find_package(Sonotide CONFIG REQUIRED)`,
see [Build details](/en/start/build).

<Steps>
  <Step title="Configure the build environment">
    First, generate the CMake cache using the pre-configured `msvc-x64-debug` preset. This preset guarantees all tests and examples are properly wired up.
    
    ```bash
    cmake --preset msvc-x64-debug
    ```
    
    <Info>If `cmake` is not in `PATH`, run the CMake binary from your installed toolchain instead.</Info>
  </Step>

  <Step title="Build the framework">
    Next, compile the library, tests, and examples.
    
    ```bash
    cmake --build --preset msvc-x64-debug
    ```
  </Step>

  <Step title="Run the test suite">
    Verify the framework core logic by running the unit tests using `ctest`.
    
    ```bash
    ctest --preset msvc-x64-debug
    ```
    
    <Warning>If you are exploring the build tree manually without the preset, you must pass `-C Debug` to CTest to avoid "No tests were found" errors.</Warning>
  </Step>

  <Step title="Try the examples">
    If the tests pass, you are ready to hear some audio! Run the compiled examples from the build directory:

    **List available audio endpoints:**
    ```bat
    build\msvc-x64-debug\examples\Debug\sonotide_list_devices.exe
    ```
    
    **Play a test sound (silence test):**
    ```bat
    build\msvc-x64-debug\examples\Debug\sonotide_render_silence.exe
    ```

    **Test the high-level playback session:**
    ```bat
    build\msvc-x64-debug\examples\Debug\sonotide_playback_session.exe "C:\Windows\Media\notify.wav"
    ```
  </Step>
</Steps>
