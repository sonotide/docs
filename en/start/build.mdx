Sonotide simplifies compilation by centralizing the setup inside CMake presets. This approach guarantees consistency whether you're building directly from the command line, Visual Studio, or a CI environment.

## The Recommended Workflow

Sonotide is optimized for the `msvc-x64-debug` preset. 

<CodeGroup>
```bash With standard PATH
# If cmake is available everywhere:
cmake --preset msvc-x64-debug
cmake --build --preset msvc-x64-debug
ctest --preset msvc-x64-debug
```

```bat Using VS Command Prompt
# Alternatively, use the full path to the Visual Studio native binary:
"C:\Program Files\Microsoft Visual Studio\18\Professional\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe" --preset msvc-x64-debug
"C:\Program Files\Microsoft Visual Studio\18\Professional\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe" --build --preset msvc-x64-debug
```

```bash From WSL integration
# Building Windows binaries straight from WSL logic:
"/mnt/c/Program Files/Microsoft Visual Studio/18/Professional/Common7/IDE/CommonExtensions/Microsoft/CMake/CMake\bin\cmake.exe" --preset msvc-x64-debug
"/mnt/c/Program Files/Microsoft Visual Studio/18/Professional/Common7/IDE/CommonExtensions/Microsoft/CMake/CMake\bin\cmake.exe" --build --preset msvc-x64-debug
```
</CodeGroup>

<Info>
  **Why do presets matter?**  
  The preset binds the generator, the multi-config build tree, and the test configurations together. This entirely avoids the classic issue where CMake generates a build tree, but CTest has no idea which configuration (`Debug` vs `Release`) it's supposed to execute.
</Info>

## Manual CTest Execution

If you prefer navigating to the directory and invoking `ctest` directly without presets, you must specify the build tree and explicitly select the configuration type:

```bash
ctest --test-dir build/msvc-x64-debug -C Debug
```

## Troubleshooting Checklist

If you encounter issues during the build process, verify the following:

- Run `cmake --version` to ensure your toolchain is recognized.
- Run `where cmake` (Windows CMD) or `which cmake` (WSL/Bash) to confirm the executable path.
- Check `git status --short` to ensure you don't have dangling or conflicting modified files.
- Always try running one of the built examples directly, like `build\msvc-x64-debug\examples\Debug\sonotide_list_devices.exe`, to verify runtime dependencies.
