## Why Choose Sonotide?

Windows audio development often devolves into a messy web of device discovery callbacks, stream initialization, format negotiation, and device loss handling. **Sonotide changes the game** by consolidating these repetitive tasks into a robust, reusable framework.

<CardGroup cols={2}>
  <Card title="Modern C++20 Design" icon="bolt">
    Employs modern semantics, explicit typing, and a robust `result<T>` error model instead of exceptions.
  </Card>
  <Card title="Flexible Stream API" icon="wave-square">
    Native support for render, capture, and loopback streams built directly on top of WASAPI.
  </Card>
</CardGroup>

## Architecture at a Glance

<div className="sonotide-panel">
  <p>
    Instead of monolithic backends, Sonotide provides modular, explicit types:
    <a href="/en/api#api-runtime"><code>runtime</code></a>, <a href="/en/api#api-streams"><code>render_stream</code></a>, and <a href="/en/api#api-playback"><code>playback_session</code></a>. Everything works together seamlessly.
  </p>
</div>

1. **Initialization:** Create a `runtime` to negotiate formats and discover devices.
2. **Streaming:** Open explicit `render_stream` or `capture_stream` objects.
3. **Playback:** Move up to a `playback_session` if you need high-level transport controls and effects out-of-the-box.
