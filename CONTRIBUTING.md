# Contributing to Sonotide docs

Thank you for improving the documentation.

## How we write here

- Use the product name `Sonotide` consistently.
- Write for an independent framework, not for a sample app.
- Keep headings in sentence case.
- Use short paragraphs and concrete examples.
- Keep the English and Russian trees structurally aligned.

## Before you send changes

- Run `mint dev` and verify the pages render.
- Check links and page paths.
- Keep branding changes in `docs.json`, `styles.css`, `logo/`, or `favicon.svg`.

## What to avoid

- Template filler or placeholder content.
- Duplicate pages that describe the same concept differently in each language.
- Any content that invents API behavior not present in the Sonotide codebase.
